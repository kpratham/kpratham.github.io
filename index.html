BAJA

Hyderabad, India

SAE Baja, FMAE Baja

Problem Statement : Develop a driver-pit communication system for a 4km long off-terrain racing track and perform realtime dynamic analysis of the buggy.

Proposed Solution : Use of IOT and XBee

I have a two year experience leading the electronics dept. of our college Baja team, Team Spardhak. The main focus was to get a quick grasp of subsystems like braking, power-transmission, suspension and then help improve the performance of the buggy in the testing and designing phase by using electronics. My team worked on various sensors, right from infrared obstacle sensors to GPS sensors, strain guages, etc. Load distribution in the buggy chasis was analysed by strain guages. We also used MPU6050 gyroscope sensor to find out the optimum camber, caster and toe angles for the wheels (For events like acceleration, braking and endurance) during the testing phase. The driver pit communication was implemented using XBee protocol.

Shop Automation System

INK Makers

t-hub, Hyderabad

Problem Statement : Most of the supermarkets these days face the problem of slow and time consuming billing process. Is there some way we could remove the whole 'standing in a queue for your turn' in the billing process?

Proposed Solution : ATM analogy

When we draw cash from an ATM, we just enter the amount we need. The whole process of counting money is done inside the machine and the user just has to collect the cash as output. Our team thought of using this same concept for small supermarkets. We presented a working model of a system where users just had to select their items on a screen, meanwhile the robots inside performed the difficult task of going through the whole warehouse to get back the desired item. The inventory had a coordinate system which sent the location of the item to the robot. The robot then goes directly to that location without spending unnecessary time wandering the supermarket looking for the product. Simultaneously, multiple robots functioned working for different customers. The billing is done as soon as the robot puts everything in the cart, and the user gets notified about his order.

Cross-Age Face Recognition using DL

Seoul, S. Korea

CVPR 2019

Problem Statement : All the current systems can be used for recognizing people only at same age. We developed a system to keep an eye on human trafficking.

Proposed Solution : Facenet

Facenet (Tensorflow implementation of OpenFace) was used to identify and recognize people in a video frame. All the current systems can be used for recognizing people only at same age. We developed a system to keep an eye on human trafficking. By fine-tuning the current available Facenet code with Large-Age-Gap dataset we tried to match missing people in a cctv footage.

Gesture Control

National Institute of technology, Warangal

EA & HAM

Problem Statement : Controlling robots in industries

Proposed Solution : Processing with MPU6050

In many industries, robots can be used to perform certain tasks wherever the working conditions are hazardous for humans. These robots need to be controlled remotely to help them navigate. We used Processing software to create a virtual workspace and then get gyroscope and accelerometer values from MPU6050 and send them wirelessly to our robot. The robot motion can be customised for numerous sets of hand motions.

Robotic exoskeleton to cure paralysis

National Institute of technology, Warangal

Final Year Project

Problem Statement : Making exoskeleton smarts by giving them the ability to self-learn.

Proposed Solution : Provide a second brain to the machine by using RL

Most of the neuro-rehabilitation therapies existing today are simple but heavy machines which help paralysis patients by enabling to the same set of exercises again and again. This is a long process and it still can't guarantee results in the end. We wanted to make this process less stressful for the patients by enabling the machine to self learn using Reinforcement learning models. Also, there was a feedback system which recorded the outcome of the program at every stage enabling to analyse if the process is benefitting the disabled.
